# Vanessa Enis — Claude Code for non-technical people

## Who

- Based in Perth (WA).
- Mechanical engineer → management consultant → product/delivery at
  Mechanical Rock.
- Self-described non-technical Claude Code user.

## The talk

Journey: Lovable → Cursor → Claude Code terminal (now exclusive).

Uses Claude Code for rapid prototypes. Tells Claude the *problem* rather
than specifying features — e.g. "Can you give me suggestions on what I
might want to see?" — and lets the first pass shape the second
conversation.

Has inverted the design-thinking sequence. Old order: brainstorm →
wireframes → dev → iterate for weeks. New order: prototype first with
Claude Code → show client → *then* brainstorm and iterate.

Connected Claude Code to Figma MCP (read-only at time of talk) and to a
PowerPoint MCP for generating decks.

As a product person on engineering teams, she connects Claude Code to
client GitHub repos and interrogates the code with product-level
questions — assessing how closely the implementation tracks the roadmap
or requirements, not reading line-by-line.

## Quotes

## Takeaways for the site

- The "personal `.md` file" is a clean newsletter hook. Lia's MEMORY
  system is already close to this, but readers haven't heard of the
  pattern.
- "Prototype before brainstorm" belongs in L2→L3 content: the moment you
  realise Claude Code is fast enough that shipping beats debating.
- The "non-technical person interrogating a codebase" workflow is an L3
  use case that generalises well — PMs, designers, exec sponsors all
  want this.
