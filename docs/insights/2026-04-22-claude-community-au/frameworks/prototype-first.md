# Prototype-first design thinking

Owner: Vanessa Enis

## The claim

Invert the design-thinking sequence. Old order: brainstorm → wireframes
→ dev → iterate. New order: prototype first with Claude Code → show
client → brainstorm and iterate.

Claude Code collapses the cost of the first prototype to minutes, which
means the cheapest place in the process has moved.

## Why it works

The old order assumes building is expensive, so you spend budget on
discussion to avoid building the wrong thing. When building is
near-free, discussion is the expensive step. A working prototype
exposes real reactions that no amount of brainstorm can surface:

- Which parts of the idea clients actually care about.
- Which "must-haves" are inert once visible.
- Which details nobody thought to mention until they saw them.

## The Vanessa pattern

1. Tell Claude the *problem*, not the features. "Can you give me
   suggestions on what I might want to see?" is a legitimate first
   prompt.
2. Ship the first prototype the same session.
3. Show the client.
4. Now brainstorm — but against real reactions, not imagined ones.
5. Iterate.
